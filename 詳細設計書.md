# Metashape 360 - 混合マルチアセット・アライメント詳細設計書

本設計書は、全天球（360°）パノラマ動画の広域データと、一般のシングルカメラ動画・高解像度写真を同一の3D空間に融合（アライメントマージ）させ、NVIDIA RTX 3050 Ti (4GB VRAM) 環境下で軽快かつ、クラッシュさせずに変換計算を実行するためのシステム設計書です。Android 等のモバイル、低スペック環境での体験まで網羅しています。

---

## 1. 画面仕様 ＆ ユーザー体験 (UX)

本アプリケーションは、1画面（シングルビュー）構成の中に、多機能で美しいコントロール、3Dプレビュー、そしてAIアシスタントを統合した **「ゴッドモード（究極のUX）」** デザインを採用しています。

### A. インプット ＆ 複数アセット管理スロット
- **多様なフォーマットの同時処理**: 
  - 360度全天球（正距円筒図）動画（Insta360等）
  - 通常の単一動画（スマホ撮影など）
  - 高解像度スナップ写真（一眼レフ等）
- **動的アセット管理**:
  - 各アセットは個別のカードとして表示。
  - アセットごとに「オリジナル解像度」「FHD(2K)」「HD(1K)」の処理限界をプルダウンで個別に制御可能。
  - アセット追加・削除ボタンが完全に稼働。
  - Android タッチ端末でも 44px のタップ領域を完全に確保し、快適に操作可能。

### B. 3050 Ti (4GB VRAM) 合併保護メーター ＆ リアルタイムプロファイラ
- NVIDIA RTX 3050 Ti 4GB VRAMのハードウェア制限に基づき、予測計算プログラムを常時回して消費GPUメモリ量を見積もる革新的メーター。
- **予測メモリゲージ表示**:
  - `4.0GB`（実質限界）に対するプロットを計算。
  - **安全帯 (グリーン: 〜3.0GB)**: クラッシュの心配なく変換可能です。
  - **警戒帯 (イエロー: 3.1〜3.6GB)**: 高い負荷。タスクマネージャーの確認を推奨。
  - **限界危険 (レッドネオン警告: 3.7GB〜)**: VRAMクラッシュの危険。自動ダウンサンプルの適用、FPS/特徴点制限の引き下げを推奨。
- **防衛モードトグル (is3050TiProfile)**:
  - ONにすると、適合特徴点数 (pointLimit) は最大 `20,000` 点に自動クランプ。
  - キーフレーム展開は最大 `2 fps` 以下に制限。
  - アセット読み込み解像度の乗数が `0.68x` に縮小され、変換作業が4GBの枠内に確実に収まります。

### C. 混合アライメント ＆ MVSプログレス (Step 1)
- 混合アセットにふさわしい階層的な進捗ロジックをシミュレート：
  1. `Extracting`: 全てのアセットから順次適切な間隔でフレームを切り出し展開。
  2. `Aligning (SIFT)`: 360パノラマ由来の「広域特徴」とシングル画像の「部分ディテール」を共通する **「共視境界 (Tie Points)」** からアライメント相関、適合を確立。
  3. `Reconstructing (MVS)`: COLMAPとMVSフュージョンを結び、ポーズ軌道とネオン3Dを生成。
- **実時間アライメント実行ロードマップ (詳細進行タグ・バッジ群)**:
  ユーザーが現在の内部処理ステータスを誤解なく一目で把握できるよう、アライメント全体を**9つの詳細な実行段階タグ（バッジ）**として可視化します：
  - `[1] FRAME_EXTRACT` (ビデオフレームのデコード＆キャッシュ展開)
  - `[2] SIFT_FEATURES` (SIFT/DoG不変特徴量の抽出・記述)
  - `[3] PAIR_MATCH` (複数カメラ相互のコヴィジビリティマッチング)
  - `[4] RANSAC_FILTER` (エピポーラ RANSAC 幾何誤合致ノイズ除外)
  - `[5] Spherical Align` (ASA: 360°広域チャンク独立アライメント)
  - `[6] Pinhole Align` (ASA: 通常カメラ/ディテール写真独立アライメント)
  - `[7] Chunk Fusion` (ASA: スケール自動連動(applyScaleTweak)チャンク統合)
  - `[8] MVS_DENSE` (OpenCLピクセル深度・MVS高密度クラウドアライメント)
  - `[9] Downsample` (WebGL表示過負荷防止・3D空間等分散ダウンサンプリング)
  これらは現在の進行状態に合わせて **「未実行(消灯)」「実行中(ネオン点滅)」「完了(緑チェックマーク付き常時点灯)」** の3状態へダイナミックに変移します。
- **リアルタイム進捗・診断モニター (可視化パネル)**:
  - **ライブアライメント処理診断コンソール**: 特徴点検出、ペアワイズマッチング、RANSAC検定、Bundle Adjustment、MVSデンス処理の各ステップが、タイムスタンプ付きのログとして1行ずつリアルタイムに流れるように出力されます。最新行への自動追従スクロールに対応。
  - **動的ペア対比ステータス**: 現在マッチング処理を行っているアセットAとアセットBの組み合わせを可視化。一致した共視特徴点の点数と、検証進捗状況（照合中・ノイズ除外中・解決完了）をカードメーターでダイナミックに増加表示。

### D. 3D WebGL ビューワー ＆ エクスポート
- スカイブルー（広域パノラマポイント）、アンバーゴールド（部分スマホポイント）、ネオングリーン（適合 Tie Points）が重なり合う美しい 3D 点群とメッシュを表示。
- COLMAP形式のカメラパラメータ群（OPENCV歪み ＆ PINHOLE）のエクスポートを網羅。

## 2. 状態管理 (State Management)

### A. `assets` (Array of `InputAsset`)
- アプリで読み込まれるアセット（360°動画、通常動画、高画質写真）のリアルタイム配列。
```typescript
export interface InputAsset {
  id: string;
  name: string;
  type: 'video_360' | 'video_single' | 'image_single';
  file: File | null;
  url: string;
  duration: number;
  frameCount: number;
  resolution: 'original' | 'fhd' | 'hd';
  extractedImages: string[];
}
```

### B. `is3050TiProfile` (Boolean)
- 3050Ti (4GB) セーフティ制限のON/OFF
- これが有効な時、パラメータ調整スライダーでの設定上限を自動的にクランプし、リアルタイム予測メモリ消費を限界値以下に押し止めます。

### C. `project` (Project)
- ワークスペース全体の構築フェーズ (`idle` | `extracting` | `aligning` | `reconstructing` | `completed`) と、展開されたフレーム総数、適合 Tie Points 数のリアルタイム反映。

## 3. モバイル・Androidへの操作性（触覚最適化）

- **Navigator Haptic Vibrate (`triggerVibrate`)**:
  - Android スマートフォン等での操作に合わせた触覚振動フィードバックを搭載。
  - シーン選択、アセット削除、アライメント開始、変換完了時に気持ちの良い振動パターン（ミリ秒指定）を発火させ、ユーザーが処理の状況を指先で把握可能。
- **レスポンシブファースト**:
  - 指でシームレスにスクロールできるよう、サイドバーを `max-h`・`overflow-y-auto` で囲い、スクロールバーを薄型に。
  - プルダウンやトグルのタップサイズを大きくし、誤操作を防止。

## 4. 例外ハンドリング ＆ クラッシュ防衛

1. **VRAMオーバー限界警告**:
   ユーザーが大量のアセットを詰め込んだり、オリジナル解像度に指定して3.6GB以上の負荷に達した場合、メーターがレッドに変化し警告メッセージがポップアップします。
2. **ビデオ読込フォールバック**:
   ブラウザでサポートされていない高プロファイル形式（例えば一部の HEVC/H.265 での4GB大容量RAW動画など）の読み込み失敗時には、ビデオ要素内部の `onerror` をキャッチし、H.264 MP4 への事前変換をアドバイスする分かりやすいトースト通知が機能します。
3. **点群データ過負荷保護**:
   ThreeViewer 内で描画されるポイント数は、アセットの解像度とFPSから決定された数（最大 6,200 点）に適切に間引かれ、ブラウザプロセスのフリーズを絶対に引き起こさない保護設計になっています。

## 5. アライメント処理順遵守 ＆ ステップロックと遷移制御

1. **計算進行中のナビゲーションロック**:
   特徴点アライメント、MVS 3D再構築、WebGLサンプリングなどの一連の重い変換計算を実行/処理中（`isProcessing === true`）である期間は、上部のステップ切り替えナビゲーター（Stepper）のクリック操作を制御（非活性/ロック）します。
   ユーザーが強制クリックを試みても遷移はブロックされ、「計算処理が実行中です。完了までお待ちいただくか、処理をキャンセルしてください」という案内ログと警告をハプティクス振動付きで発火させ、アライメントが不完全な状態で他画面に流出する事故を遮断します。
2. **完了時の自動画面遷移の廃止 ＆ 確認誘導**:
   アライメントが最後まで完全に終了した瞬間に、システムが勝手に `setActiveStep(2)` を発火させるのを廃止します。
   代わりに、完了後にStep 1のアライメント画面最上部 / またはコンソール周辺に「🎉 混合アライメント ＆ MVS 3D再構築が正常に完了しました！」という華やかな完了ダイアログを優雅に出現させ、「COLMAPエクスポート（Step 2）へ進む ➔」の専用アクションボタンを設置。ユーザー自身がボタンを押すか、上部Stepperを使って自律的にエクスポートおよび3Dビューワーへ遷移できるように改良します。これにより、処理完了と次の成果物(COLMAPパラメータ)の確認との間の体験を自然に連結します。

## 6. 高精度3DアライメントZIPエクスポート ＆ 各種SFM・フォトグラメトリツール高速連携

1. **RealityScanの制約クリアと連携設計**:
   - スマホ用RealityScanアプリは「撮影・直通クラウド計算」に特化したソフトであるため、スマホ単体では外部アライメント（COLMAP TXTなど）をインポートできません。
   - 本システムではこの制約を徹底解決するため、クラウドで自動連携されるPCデスクトップ版 **RealityCapture**、および業界標準の **Agisoft Metashape** との高速な位置情報連携ルートを確立。
2. **一括ZIPパッケージ出力 (`jszip` を使用)**:
   - ボタン一つで次のファイルをまとめて一括ZIP圧縮し、ダウンロード可能にします。
     - `cameras.txt`, `images.txt`, `points3D.txt` (COLMAP 構成ファイル)
     - `realitycapture_alignment.xml` (RealityCapture 等で空間構成を瞬時にロードしてアライメントをスキップできるXMLコンポーネント)
     - `metashape_alignment.xml` (Agisoft Metashape カメラインポート準拠XML)
     - `camera_alignment_trajectory.csv` (一般的なCSV軌道。X, Y, ZおよびQW, QX, QY, QZ回転、Yaw/Pitch/Roll)
     - `ReadMe_PHOTOGRAMMETRY_GUIDE.txt` (アライメント未処理での分裂・位置解離エラーを防ぐインポート手順書)
3. **UI/UXでの状況親切説明**:
   - ユーザーが「なぜスマホのRealityScanにCOLMAP textファイルを読み込めないのか」迷わないよう、UI内に **「連携ガイダンス」** エリアを新設し、問題の原因と具体的な解決手順を優しく提示します。
